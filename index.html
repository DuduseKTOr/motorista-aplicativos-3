📚 Tutorial — Configurar Drive

O Google exige que cada app tenha sua própria credencial. Configure uma vez, use pra sempre. Leva ~10 minutos.

1. Criar projeto no Google Cloud

  1. Acesse: console.cloud.google.com
  2. Faça login com a conta Google onde quer salvar os backups
  3. No topo, clique no seletor de projeto → NOVO PROJETO
  4. Nome: Motorista Backup → criar
  5. Aguarde alguns segundos e selecione o projeto recém-criado

2. Habilitar Drive API

  1. No menu lateral (☰): APIs e serviços → Biblioteca
  2. Busque Google Drive API
  3. Clique no resultado → ATIVAR

3. Configurar tela de consentimento

  1. Menu lateral: APIs e serviços → Tela de consentimento OAuth
  2. Tipo de usuário: Externo → CRIAR
  3. Preencha:
    • Nome do app: Motorista Backup
    • Email de suporte: seu email
    • Email do desenvolvedor: seu email
  4. SALVAR E CONTINUAR (pode pular as próximas telas)
  5. Na tela Usuários teste: clique ADD USERS → adicione seu próprio email Google → SALVAR

4. Criar credencial OAuth Client ID

  1. Menu: APIs e serviços → Credenciais
  2. Topo: + CRIAR CREDENCIAL → ID do cliente OAuth
  3. Tipo: Aplicativo da Web
  4. Nome: Motorista App
  5. Em Origens JavaScript autorizadas clique em + ADICIONAR URI e cole a URL do seu app no GitHub Pages. Exemplo:
    https://SEUUSUARIO.github.io
    Não inclua o caminho /motorista/ no final, só o domínio.
  6. Clique CRIAR
  7. Aparece um popup com o Client ID. Copie ele inteiro (algo tipo 1234567890-abc...xyz.apps.googleusercontent.com)

5. Colar no app

  1. Feche este tutorial
  2. Clique em ⚙️ CONFIGURAR
  3. Cole o Client ID e salve
  4. Clique CONECTAR DRIVE
  5. Vai abrir popup do Google → escolha sua conta → autorize
  6. ⚠️ Pode aparecer "Google ainda não verificou este app" — clique Avançado → Continuar para Motorista Backup. Isso é normal (é seu próprio projeto, não precisa de verificação)
🔒 Privacidade: O app só acessa arquivos que ele próprio cria no seu Drive (escopo drive.file). Não vê seus outros arquivos.
